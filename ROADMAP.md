# Roadmap

This roadmap describes the public direction for Project V26. Items may change as gameplay and backend systems mature.

## Near Term

- Improve first-session onboarding and the new player route.
- Continue polishing the home dashboard: News, Shouts and player progress.
- Strengthen settings UX for profile, password, language, theme, notifications and privacy.
- Keep planned features visually separate from backend-supported features.
- Improve Docker/local setup documentation and first-run troubleshooting.

## Mid Term

- Expand notification preference integration.
- Improve DM privacy and block-list workflows when backend support is complete.
- Add safer development fixtures for public contributors.
- Improve newspaper and public content moderation tools.
- Review legacy frontend asset tooling and document the supported path.
- Add contributor-friendly setup checks once the public development flow is stable.

## Long Term

- Harden admin workflows and audit logging.
- Improve accessibility and responsive behavior across core screens.
- Add contributor-friendly automated checks.
- Prepare a stable public demo build.
- Document deployment and backup guidance for maintainers.

## Not Planned for Public Build

- Committing production secrets or private data.
- Shipping database dumps with real player data.
- Adding UI that pretends unsupported backend features are active.
